Lista B – Questões 1 a 10

1. Código aberto vc Código Fechado

  • Sistema Operacional de Código Aberto: Permite que qualquer pessoa veja, modifique e distribua seu código (ex: Linux, FreeBSD, Android).
  • Vantagens: Personalização e segurança.
  • Desvantagens: Suporte limitado e complexidade.
  • Sistema Operacional de Código Fechado: Não permite acesso ao código, sendo controlado pela empresa (ex: Windows, macOS, iOS).
  • Vantagens: Suporte profissional e facilidade de uso.
  • Desvantagens: Custo e pouca flexibilidade.

2. Análise Comparativa de Instalação: Windows vs Ubuntu

A. Licenciamento

  • Windows: Pago, código fechado, uso limitado pela licença da Microsoft.
  • Ubuntu: Gratuito, código aberto (GNU/GPL), sem chave de ativação.

B. Personalização Durante a Instalação

  • Windows: Personalização básica.
  • Ubuntu: Opções avançadas como codecs, drivers, criptografia e particionamento.

C. Necessidade de Drivers

  • Windows: Suporte básico, mas drivers específicos geralmente devem ser baixados.
  • Ubuntu: Muitos drivers incluídos, mas pode exigir ativação de drivers proprietários.

3.

  • A escolha entre o modo texto (CLI) e a interface gráfica (GUI) depende das necessidades do usuário e do tipo de tarefa.
  • Modo Texto (CLI):
    • Alta eficiência e velocidade em tarefas repetitivas.
    • Permite automação via scripts (ideal para administradores).
    • Menor consumo de recursos (bom para máquinas antigas ou servidores).
    • Desvantagens: exige conhecimento técnico e é pouco visual.
  • Interface Gráfica (GUI):
    • Fácil de usar e intuitiva, com menus, ícones e janelas.
    • Ideal para tarefas visuais: edição de imagens, navegação, etc.
    • Feedback visual imediato e suporte a multitarefa.
    • Desvantagens: consome mais recursos e tem menor capacidade de automação.

O que são Diretórios?

Diretórios de computadores são pastas usadas para organizar dados e informações armazenadas em discos rígidos ou outras mídias de armazenamento.

Diretórios Comuns no Windows

  • C:\Windows: Contém os arquivos essenciais do sistema operacional, como bibliotecas (DLLs), drivers, arquivos de inicialização e ferramentas do sistema.
  • C:\Program Files e C:\Program Files (x86): Armazenam os programas instalados no sistema. A versão (x86) é usada para aplicativos de 32 bits em sistemas de 64 bits.
  • C:\Users: Contém os perfis dos usuários, documentos, downloads, configurações pessoais e arquivos individuais de cada conta.

Diretórios Comuns no Linux

  • /bin: Armazena comandos essenciais do sistema, como:
    • ls – Lista os arquivos de um diretório e suas informações
    • cp – Copia arquivos de um local para outro
    • mv – Move ou renomeia arquivos e diretórios
    • bash – Interpretador de comandos utilizado para controlar o sistema operacional
  • /etc: Contém arquivos de configuração do sistema e de aplicativos, como /etc/passwd e /etc/fstab.
  • /home: Diretório pessoal dos usuários, onde ficam os documentos, configurações individuais e arquivos pessoais.
  • /var: Armazena arquivos variáveis, como logs (/var/log), caches e bancos de dados temporários.

5. Arquivos Fechado vs Aberto

  • Função: O gerenciamento de usuários e permissões controla quem pode acessar ou modificar arquivos e recursos no sistema.
  • No Windows (código fechado): Utiliza o sistema de arquivos NTFS, que permite definir permissões detalhadas para usuários e grupos.
  • No Linux (código aberto): Usa permissões baseadas em três níveis (rwx): dono, grupo e outros, controlando leitura, escrita e execução.

6.

  • O processo de inicialização começa com o fornecimento de energia e o POST (Power-On Self Test), que verifica o funcionamento básico do hardware.
  • Se o POST for bem-sucedido, o BIOS/UEFI localiza o dispositivo de boot e carrega o bootloader (GRUB no Linux, Windows Boot Manager no Windows).
  • O bootloader carrega o kernel do sistema operacional na memória, iniciando a transição do hardware para o software.
  • O kernel detecta e inicializa os componentes via drivers, monta o sistema de arquivos raiz e inicia processos essenciais (systemd no Linux, Session Manager no Windows).
  • Por fim, o sistema carrega os serviços de usuário, gerenciador de login e interface gráfica ou terminal.
  • Diferenças:
    • Linux (código aberto): Altamente personalizável em todas as etapas do boot.
    • Windows (código fechado): Processo padronizado e simples, mas com menos opções de personalização.
    • Em recuperação de sistema, o Linux oferece ferramentas e logs detalhados; o Windows é mais limitado nesse aspecto.

7. Compartilhamento, Mapeamento

a) O que são unidades e pastas em rede?

Pastas em rede são compartilhadas por outros dispositivos. Unidades em rede são pastas mapeadas que aparecem como discos (ex: Z:).

b) O que é "compartilhamento" e "acesso"?

  • Compartilhamento: Disponibilizar um recurso na rede.
  • Acesso: Permissões como leitura, gravação ou nenhuma.

c) O que é mapeamento?

É atribuir uma letra de unidade a uma pasta da rede, facilitando seu acesso.

A. Definição de Segurança Cibernética

Segurança Cibernética é o conjunto de práticas, tecnologias e processos utilizados para proteger sistemas, redes, dispositivos e dados contra acessos não autorizados, ataques, danos ou roubo. Seu objetivo é garantir a confidencialidade, integridade e disponibilidade das informações no ambiente digital.

B. Tipos Comuns de Ameaças Cibernéticas

  • Malware: Software malicioso (vírus, worms, trojans) projetado para causar danos ou obter acesso não autorizado.
  • Phishing: Técnica de engano por e-mails ou sites falsos para obter informações sensíveis.
  • Ransomware: Criptografa dados da vítima e exige pagamento para restaurar o acesso.
  • Ataques DDoS: Sobrecarga de tráfego para tornar um serviço ou rede indisponível.
  • Spyware: Programa que monitora o usuário sem consentimento, coletando dados pessoais.

C. Vulnerabilidades em um Sistema

Vulnerabilidades são falhas ou fraquezas em sistemas, aplicativos ou redes que podem ser exploradas por cibercriminosos.

  • Erros de configuração
  • Falhas de programação (bugs)
  • Sistemas desatualizados
  • Falta de boas práticas de segurança

Essas falhas podem resultar em roubo de dados, acesso indevido ou falhas operacionais.

D. Importância da Proteção de Credenciais

Credenciais são dados usados para autenticação (nome de usuário, senha, token, certificado digital).

  • Evita acessos não autorizados
  • Garante a segurança de dados sensíveis
  • Protege contra fraudes e perdas financeiras

Práticas recomendadas: senhas fortes, autenticação multifator e não compartilhar credenciais.

E. Engenharia Social e Exemplos

Engenharia Social é a manipulação psicológica para obter informações confidenciais ou induzir ações inseguras.

  • Telefonema falso de suporte técnico: Criminoso finge ser da TI e obtém senhas ou instala malware.
  • E-mail com urgência falsa: Finge ser um superior solicitando clique em links ou envio de dados sigilosos.

9.

Recomendações de Segurança para Usuários Comuns

  • Manter programas e antivírus sempre atualizados
  • Utilizar senhas fortes com autenticação em dois fatores
  • Evitar clicar em links ou fazer downloads de fontes suspeitas
  • Baixar somente de sites confiáveis
  • Realizar backups regularmente
  • Salvar cópias de arquivos importantes no Google Drive, HD externo ou pendrive

Recomendações para Administradores de Sistema

  • Desativar serviços e recursos que não estão sendo utilizados
  • Controlar o acesso de usuários aos recursos do sistema
  • Conceder apenas as permissões necessárias (ex.: usuários comuns não devem ter acesso de administrador)
  • Manter o firewall do sistema sempre ativado
  • Utilizar o firewall para bloquear tentativas de invasão

Função de um Firewall Nativo do Sistema Operacional

  • Bloqueia acessos não autorizados: Impede que hackers ou malwares se conectem ao computador pela internet.
  • Controla quais programas usam a internet: Permite ou bloqueia conexões de aplicativos específicos.
    • Permitir: Navegador (Chrome, Firefox) acessar a web
    • Bloquear: Um jogo pirata tentando enviar dados para um servidor suspeito
  • Filtra tipos de tráfego perigosos: Bloqueia pacotes de rede maliciosos, como:
    • Ataques DDoS (muitas conexões para sobrecarregar o sistema)
    • Port scanning (tentativas de descobrir portas abertas)
  • Registra tentativas de invasão: Gera logs (registros) que podem ser analisados posteriormente

10. Pontos Críticos Sistema Operacional

  • 1. Gerenciamento de Usuários e Permissões: Define quem pode acessar ou modificar arquivos e sistemas, essencial para a segurança.
  • 2. Configuração de Rede: Ajusta IP, DNS e firewall, garantindo conectividade e proteção contra acessos indesejados.
  • 3. Atualizações e Patches: Mantém o sistema seguro e estável, corrigindo falhas e melhorando o desempenho.
  • 4. Serviços e Processos: Controla quais programas iniciam com o sistema, otimizando recursos e evitando vulnerabilidades.
  • 5. Políticas de Segurança: Define regras como senhas, criptografia e bloqueio de dispositivos, protegendo dados e usuários.
